Pintando con el canvas, javascript.

Yone Moreno Jiménez

·

Follow

7 min read
·
Apr 1, 2018

SIguiendo el curso de Wes Bos, hoy toca usar el canvas. Es un lienzo sobre el cual interactuar, en concreto podemos pintar sobre su contexto, que viene a ser el medio mediante el cual dibujamos, como si se tratase del papel y la pintura que ponemos sobre el lienzo.

El resultado final será un programa con el cual dibujar, y a medida que hacemos el trazo se cambia el color y ancho del mismo:

Lo primero que tenemos es la etiqueta canvas en el html:

<canvas id="draw" width="800" height="800"></canvas>

Por tanto lo primero que hacemos es cogerlo:

const canvas = document.querySelector('#draw');

Luego queremos acceder al contexto, el cual es el medio con el cual pintar, lo hay de varios tipos 3d para videojuegos, 2d:

const ctx = canvas.getContext('2d');

Lo primero que necesitamos es estirar el canvas para que ocupe todo el ancho y alto de la ventana, ya que ahora está a 800x800:

Para ello:

canvas.width = window.innerWidth;
canvas.height = window.innerHeight;

Con lo que se ajusta:

Además tenemos que configurar el estilo. Para ello usamos strokeStyle para el color, lineJoin para decir como queremos que se unan dos líneas y lineCap para decir cómo dibujar el fin de una línea:

ctx.strokeStyle = '#BADA55';
ctx.lineJoin = 'round';
ctx.lineCap = 'round';

Es importante saber cuándo se está en modo dibujado. La idea es que si pulsamos el ratón y arrastramos estamos dibujando y si dejamos de pulsar podemos mover libremente sin pintar:

let isDrawing = false;

Necesitamos guardar la posición del cursor en todo momento:

let lastX = 0;
let lastY = 0;

Luego detectamos el movimiento del ratón en el canvas y mostramos en la consola el evento:

function draw(e) {
    console.log(e);
}

canvas.addEventListener('mousemove', draw);

Dándonos una lista de posiciones del cursor al moverlo sobre el canvas:

Además en esos eventos tenemos la posición del cursor como: offsetX, offsetY:

Después, queremos que el usuario dibuje cuando está presionando el ratón y moviéndolo por el canvas, para ello usamos la variable isDrawing para evitar que se dibuje cuando solo se mueve el ratón por pantalla:

function draw(e) {
    if (!isDrawing) return;
    console.log(e);
}

canvas.addEventListener('mousemove', draw);
canvas.addEventListener('mousedown', () => isDrawing = true);
canvas.addEventListener('mouseup', () => isDrawing = false);
canvas.addEventListener('mouseout', () => isDrawing = false);

Por tanto el evento mousedown controla la pulsación del ratón y se asegura que estamos en modo dibujo. mouseup hace la operación contraria, nos permite salir de modo dibujo cuando dejamos de pulsar el ratón. Lo interesante es el porqué de usar el mouseout. Este evento se llama cuando hacemos que el cursor salga de la ventana, y hace falta porque si no al volver a entrar estaríamos dibujando!.

Ahora empezamos a hacer la funcionalidad de dibujar.

Lo que queremos es dibujar para ello usamos beginPath() para decir que es el inicio del dibujo y stroke() para terminarlo y mostrarlo. Luego moveTo() nos permite desplazar el pincel a la posición que queramos, y lineTo() dibuja la línea desde donde está hasta el punto final:

function draw(e) {
    if (!isDrawing) return;
    console.log(e);
    ctx.beginPath();
    ctx.moveTo(lastX, lastY);
    ctx.lineTo(e.offsetX, e.offsetY);
    ctx.stroke();
}

Con lo cual conseguimos el resultado:

Como vemos siempre se empieza a dibujar desde arriba a la izquierda, y son líneas independientes. El motivo es que nuestra lastX, lastY los puntos de inicio del dibujo son 0, 0.

La solución es actualizar la posición del cursor guardada, lastX, lastY, con la actual, la presionada:

lastX = e.offsetX;
lastY = e.offsetY;

Con esto tenemos:

La posibilidad de dibujar desde otros puntos a parte de arriba a la izquierda. Eso sí, quedan dos detalles: el primero es que el dibujo se sigue empezando en 0, 0 sin importar donde cliquemos la primera vez, ya que lastX y lastY siguen estando iniciadas a 0. Lo segundo, es que estamos obligados a dibujar una línea continua. El motivo es que no tenemos en cuenta que el usuario haya dejado de dibujar, haya movido el cursor y empiece de nuevo.

Un truco, podemos asignar ambas variables a la vez:

function draw(e) {
    if (!isDrawing) return;
    console.log(e);
    ctx.beginPath();
    ctx.moveTo(lastX, lastY);
    ctx.lineTo(e.offsetX, e.offsetY);
    ctx.stroke();
    [lastX, lastY] = [e.offsetX, e.offsetY];

}

Lo que haremos para permitir dibujar y no obligar a que sea una raya continua, es actualizar las posiciones inmediatamente al presionar el ratón:

canvas.addEventListener('mousedown', (e) => {
    isDrawing = true;
    [lastX, lastY] = [e.offsetX, e.offsetY];
});

Con ello ya se mejora:

Además nos interesa cambiar el grosor de la línea con:

ctx.lineWidth = 50;

También comentar que si quitamos el que las líneas sean redondas:

ctx.strokeStyle = '#BADA55';
//    ctx.lineJoin = 'round';
//    ctx.lineCap = 'round';
    ctx.lineWidth = 50;

Se verían:

Un tema interesante es cómo cambiar los colores de la línea. Bien, podemos usar el espacio de colores hsl, cuyas siglas significan hue, tono; saturation, saturación; light, luz. En esta página se explica de forma gráfica: http://mothereffinghsl.com/

Lo interesante de este espacio de colores es que si 0 es rojo y 360 es el otro lado, el violeta, 361 volverá a ser rojo, es decir se repiten, lo cual podemos usar en el color de nuestra línea.

Lo que haremos es una nueva variable:

let hue = 0;

Por lo que dentro de draw() podremos poner el color de la línea:

function draw(e) {
    if (!isDrawing) return;
    console.log(e);
    ctx.strokeStyle = `hsl(${hue}, 100%, 50%)`;
    ctx.beginPath();
    ctx.moveTo(lastX, lastY);
    ctx.lineTo(e.offsetX, e.offsetY);
    ctx.stroke();
    [lastX, lastY] = [e.offsetX, e.offsetY];
}

En concreto, se coge el tono, hue de la variable, se pone 100% a la saturación para que sea brillante, y 50% a la luz para que sea un color intermedio.

Con ello tenemos:

Como vemos la línea siempre sale roja, porque el hue siempre está a 0.

Nos interesa que cambie el color, así que dentro de draw incrementamos en 1 cada vez que dibujamos:

hue++;

Y se verá:

Por ejemplo lo interesante es que el hue, el tono siempre se incrementa pero los colores se repiten:

También si queremos incluir que el hue esté limitado y se reinicie en draw:

if (hue >= 360) {
    hue = 0;
}

Así volverá al inicio:

Por ejemplo volvería al rojo, al 0.

Es posible hacer algo similar con el ancho de la línea, que comience en 0 y se incremente a medida que cambia el tono:

ctx.lineWidth = hue;

Quedando:

Lo que sería más interesante es que la línea se haga grande al cambiar de color y cuando llegue al violeta empiece a decrecer.

Para ello necesitamos saber en qué dirección vamos, es decir si crece, será true, y si decrece, false:

let direction = true;

Y luego subimos el tamaño en draw:

ctx.lineWidth++;

Lo que pasa es que esto hará que se engrandezca hasta ocupar todo:

Para la lógica de incrementar y decrementar el grosor de la línea necesitamos controlar su ancho y cambiar la direction cuando sea necesario:

function draw(e) {
    if (!isDrawing) return;
    console.log(e);
    ctx.strokeStyle = `hsl(${hue}, 100%, 50%)`;
    ctx.beginPath();
    ctx.moveTo(lastX, lastY);
    ctx.lineTo(e.offsetX, e.offsetY);
    ctx.stroke();
    [lastX, lastY] = [e.offsetX, e.offsetY];
    hue++;

    if (hue >= 360) {
        hue = 0;
    }

    if (ctx.lineWidth >= 100 || ctx.lineWidth <= 1) {
        direction = !direction;
    }
    if (direction) {
        ctx.lineWidth++;
    } else {
        ctx.lineWidth--;
    }
}

En concretos si el tamaño excede de 100 y pasa hacia abajo de 1 cambiamos la dirección, de forma que si es true se hace gruesa y si es false se disminuye.

También podemos hacer una ‘mezcla de colores’, mediante el ctx.globalCompositeOperation, por ejemplo si ponemos:

ctx.globalCompositeOperation = 'multiply';

Se verá:

Es decir los colores se fusionan.

Por último con este ejercico hemos aprendido a usar el canvas, los eventos del ratón, a cambiar de forma dinámica propiedades como el color y el grosor, y hemos usado condicionales para la lógica del estar en modo dibujado.
